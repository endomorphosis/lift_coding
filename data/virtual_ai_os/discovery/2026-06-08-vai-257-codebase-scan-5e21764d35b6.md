## Resolution

The exception path was a maintenance risk: `_add_to_cache` tolerated local cache
write failures, but a metadata write failure could leave an orphaned cache data
file that `_clean_cache` would not account for. The cache insert now writes data
and metadata through temporary files, catches only expected filesystem and JSON
serialization errors, removes temporary or partially committed files on failure,
and logs the failed bucket/key with traceback context.

## Resolution

The broad `except Exception as e` path could hide workflow-file read failures and
let the generator emit an incomplete workflow trigger list. Replaced it with a
typed `WorkflowListError` that is raised for unreadable files and for YAML parse
failures that cannot be recovered by the existing top-level `name:` regex
fallback.

The fallback remains intentional because the current checkout contains a workflow
that PyYAML cannot parse but GitHub Actions can still use, and the generator only
needs the top-level workflow name.

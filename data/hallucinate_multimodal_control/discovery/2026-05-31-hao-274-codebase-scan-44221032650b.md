## Resolution (HAO-274)

The `test_redundant_uppercase_ranges_removed` test at line 269 used `'XXX'` as
the replacement argument in `pattern.sub('XXX', '0xDEADBEEF')`.  This is the
old normalisation sentinel that was replaced by `'\x00'` in VAI-144 (see
`test_sentinel_is_single_null_byte_not_xxx` and
`test_message_containing_sentinel_not_falsely_similar` in the same file).

Using `'XXX'` in the sub call did not cause a functional bug because the
argument is just a dummy replacement used to check that the regex matches, but
it was misleading and inconsistent with the rest of the test suite's
documentation.

**Fix**: Changed both `pattern.sub('XXX', ...)` calls to
`pattern.sub('\x00', ...)` so the test mirrors the actual `_SIMILAR_SENTINEL`
value and cannot be confused with the stale sentinel.

Status: resolved — not a false positive.

## Resolution

**Bug**: The regex pattern `0x[0-9a-f]+` (and `ID: [a-f0-9-]+`) only matched
lowercase hex digits, so error messages containing uppercase hex addresses
(e.g. `0xDEADBEEF`) or uppercase UUID segments were not normalised, causing
`_messages_similar` to miss genuine duplicates.

**Fix** (`error_monitor.py`):
- Replaced the inline `re.sub(pattern, ...)` calls with a class-level compiled
  pattern `_SIMILAR_PATTERN` using `re.IGNORECASE` and explicit `[0-9a-fA-F]`
  ranges so both cases are scrubbed consistently.
- Compiling once avoids repeated pattern compilation on every call.

**Validation**: `python3 -m py_compile` passes.

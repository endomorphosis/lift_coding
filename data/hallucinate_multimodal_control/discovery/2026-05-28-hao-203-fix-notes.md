# HAO-203 Fix Notes

Date: 2026-05-28
Source: hallucinate_app/hallucinate_app/python/hallucinate_app/auth_keystore_integration.py:288
Kind: swallowed_exception

## Finding

`get_authorized_key_info` caught all exceptions and returned `None`, making it
impossible for callers to distinguish between an intentional `None` (unauthorized
or provider not found) and an unexpected internal error.

## Fix

Changed the `except` block at line 288 to `raise` instead of `return None`.
The exception is still logged with `logger.exception()` for observability, but is
now re-raised so callers can propagate or handle it appropriately.

Updated the docstring to document the new `Raises` section.

## Why not a false positive

The other methods in the same file (`list_authorized_providers`,
`rotate_authorized_key`, `issue_key_access_capability`) return typed sentinel
values (`None` / `False`) whose meaning is unambiguous at the call sites.
`get_authorized_key_info` returns `Optional[Dict]` where `None` already means
"unauthorized or not found", so swallowing an exception by returning `None` was
genuinely misleading.

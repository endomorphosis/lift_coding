# HAO-681 Swissknife Dashboard Catalog Consumer

Date: 2026-06-27
Task: HAO-681
Goal id: VAIOS-G723
Track: integration
Depends on: HAO-677, HAO-680

HAO-681 proves that Swissknife applications consume the Hallucinate App MCP dashboard catalog instead of maintaining a dashboard-only mock or duplicate schema. The shared catalog is `hallucinate_app.mcp_dashboard_capability_catalog.v1`, generated by `hallucinate_app.node.mcp_daemon_manager.getDashboardCapabilityCatalog`, and consumed by `hallucinate_app.swissknife.mcp_capability_registry`.

## Evidence

- `hallucinate_app/hallucinate_app/node/mcp_daemon_manager.js` exposes `swissknife_catalog_consumer_proof` in the Hallucinate App MCP dashboard catalog.
- `hallucinate_app/test/e2e/fixtures/vai-512-mcp-dashboard-catalog.json` mirrors the live catalog and names the HAO-681 Swissknife consumer receipt.
- `swissknife/test/e2e/fixtures/hao-681-mcp-dashboard-catalog-consumer.json` records the shared receipt schema and required control_surface receipt fields.
- `swissknife/test/e2e/mcp-dashboard.spec.ts` and `swissknife/scripts/test-mcp-dashboard-consumer.cjs` validate the same catalog entries used by the Hallucinate dashboard.
- `hallucinate_app/test/e2e/mcp-dashboard-interoperability.spec.ts` runs `npm --prefix swissknife run test:e2e:mcp` and checks the HAO-681 payload.

## Covered Swissknife Applications

- Swissknife storage application: `ipfs-kit-storage-console` consumes the `ipfs_kit_py` catalog entry for `tools/list` and the non-mutating `ipfs_status` `tools/call` probe.
- Swissknife dataset application: `ipfs-datasets-workbench` consumes the `ipfs_datasets_py` catalog entry for `tools/list` and the non-mutating `datasets_list` `tools/call` probe.
- Swissknife compute application: `ipfs-accelerate-console` consumes the `ipfs_accelerate_py` catalog entry for `tools/list` and the non-mutating `hardware_profile` `tools/call` probe.

## Receipt Contract

Swissknife applications require the Hallucinate dashboard receipt schema `mcp_server_invocation_receipt_v1` with `interaction_envelope`, `policy_decision`, `mediation_receipt`, `mediation_receipt_id`, `receipt_cid`, and `mcpplusplus_descriptor_evidence`. This proves that a backend feature exposed from Hallucinate App is the same backend feature Swissknife can invoke under MCP++ and control_surface mediation.

## Resolution

The reported path was the broad exception handler in `import_module_from_path`,
which logged import failures and returned `None`. The implementation now raises
import failures instead of swallowing them, rolls back partial `sys.modules`
entries on failed loads, and explicitly rejects missing module files before
creating an import spec. Startup catches the import failure, logs it with
traceback context, and exits.

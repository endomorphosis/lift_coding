# HAO-288 Fix Notes

Date: 2026-06-05
Source: hallucinate_app/hallucinate_app/python/worker.py
Kind: swallowed_exception + assignment_typo
Status: fixed

## Findings

### 1. Swallowed exception at line 89 (original)

The `__main__` guard caught all exceptions but only called `print(e)`, losing:
- The full traceback (stack frames, file names, line numbers)
- A non-zero process exit code, so callers could not detect failure

**Fix**: replaced `print(e)` with `traceback.print_exc()` and added `sys.exit(1)`.

### 2. Assignment typo at line 20 (original)

```python
self.metadata["role"] == "master"   # comparison, not assignment
```

This is a `KeyError` (empty dict) disguised as a no-op comparison. The exception
was immediately caught by the very handler this task was filed to fix, making the
bug invisible at runtime. The swallowed-exception path is what kept this silent.

**Fix**: changed `==` to `=`.

## Changes

- `hallucinate_app/hallucinate_app/python/worker.py`
  - Added `import sys` and `import traceback`
  - Line 20: `==` → `=` (metadata role assignment)
  - Lines 89-91: `print(e); pass` → `traceback.print_exc(); sys.exit(1)`

# HAO-358 Codebase Scan: Swallowed Exception in ipfs_kit_cli_ultra_fast.py:78

**Scan date:** 2026-06-08
**Task ID:** HAO-358
**Priority:** P1
**Track:** runtime

## Finding

**File:** `external/ipfs_kit/archive/cli_drafts/ipfs_kit_cli_ultra_fast.py`
**Line:** 78
**Method:** `cmd_status()` in `UltraFastCLI`

### Issue

A bare `except:` clause at line 78 silently swallowed all exceptions, including
`KeyboardInterrupt` and `SystemExit`, when reading the daemon PID file. Any
`OSError`, `ValueError` (malformed PID), or permission error was discarded with
no diagnostic output, making it impossible to distinguish a missing PID file from
a corrupt one or a permission problem.

```python
# BEFORE (line 78) – bare except swallows everything
try:
    with open("/tmp/ipfs_kit_daemon.pid", 'r') as f:
        pid = int(f.read().strip())
    print(f"   PID: {pid}")
except:
    pass
```

### Root Cause

The PID-file read block used a bare `except:` clause instead of a scoped
`except Exception` handler. Other exception handlers in the same file (lines 177,
207, 250, etc.) all use `except Exception as e:` with appropriate logging, making
this a deviation from the established pattern.

### Fix Applied

Replaced the bare `except:` with `except Exception as e:` and added a
`logger.warning()` call. Also added `import logging` and a module-level logger
instance to support the warning call.

```python
# AFTER – exception captured and logged
try:
    with open("/tmp/ipfs_kit_daemon.pid", 'r') as f:
        pid = int(f.read().strip())
    print(f"   PID: {pid}")
except Exception as e:
    logger.warning(f"Could not read daemon PID file: {e}")
```

`KeyboardInterrupt` and `SystemExit` are no longer swallowed, allowing the process
to terminate cleanly when requested.

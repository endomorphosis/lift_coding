## Resolution

Status: fixed
Date: 2026-05-28

The `list_authorized_providers` method was catching `Exception` and returning `None` on
any error, making it impossible for callers to distinguish an authorization denial (expected
`None`) from an unexpected runtime failure (also `None`). This is a real bug: callers that
rely on `None` meaning "not authorized" will silently ignore backend failures.

Fix: replaced `return None` in the except block with `raise`, so the exception propagates
to the caller after being logged. The docstring was updated to document the `Raises` clause.
`None` is now only returned on a genuine authorization denial (expected path). The exception
is still logged via `logger.exception` before re-raising so the full traceback appears in
application logs.

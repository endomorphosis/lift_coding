# HAO-289 Fix Notes

Date: 2026-06-05
Source: hallucinate_app/python/hallucinate_app/control_surface_policy.py:409

## Finding

`except Exception:` at line 409 swallowed all exceptions in a try block that only
performs a single `import` statement. The only expected failure mode is `ImportError`
(or its subclass `ModuleNotFoundError`) when `ipfs_datasets_py` is not installed.

## Fix

Narrowed `except Exception:` to `except ImportError:` so that unexpected errors
(e.g., `AttributeError`, `SyntaxError`, `TypeError`) propagate instead of being silently
swallowed. The compatibility-shim function `_install_ipfs_at_time_evaluator_adapter` is
an optional enhancement that must degrade gracefully when the upstream module is absent,
but must not hide programming errors.

## Disposition

Bug fix — not a false positive.

# HAO-315 Codebase Scan — Swallowed Exception in direct_mcp_server.py

**File:** `external/ipfs_kit/archive/applied_patches/direct_mcp_server.py`  
**Line:** 217  
**Severity:** P1  

## Finding

In `start_other_instance()`, the `except Exception` block at line 217 logs only
`str(e)` via `logger.error()`, discarding the full stack trace.  When
`subprocess.Popen` raises (e.g. `FileNotFoundError`, `PermissionError`), the
traceback context is lost and the caller receives `None` with no way to
distinguish the root cause from the logged output alone.

```python
# before
    except Exception as e:
        logger.error(f"Failed to start other instance: {e}")
        return None
```

## Fix Applied

Replaced `logger.error()` with `logger.exception()`, which automatically appends
`exc_info=True` so the full traceback is captured in the log record at `ERROR`
level.

```python
# after
    except Exception as e:
        logger.exception(f"Failed to start other instance: {e}")
        return None
```

`logger.exception()` is the idiomatic Python pattern for logging exceptions with
traceback inside an `except` block.  The return value and call-site behaviour are
unchanged; only observability is improved.

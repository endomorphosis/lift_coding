# HAO-360 Codebase Scan: Swallowed Exception in vscode_mcp_server.py:304

**Scan date:** 2026-06-08
**Task ID:** HAO-360
**Priority:** P1
**Track:** runtime

## Finding

**File:** `external/ipfs_kit/archive/legacy_servers/vscode_mcp_server.py`
**Line:** 304
**Method:** `system_health_tool` (disk usage loop)

### Issue

A bare `except:` clause at line 304 silently swallowed all exceptions, including
`KeyboardInterrupt` and `SystemExit`, when reading disk usage for individual
paths inside `system_health_tool`. Any `PermissionError`, `OSError`, or other
error from `psutil.disk_usage()` was discarded with no logging, making failures
completely invisible to operators.

```python
# BEFORE (line 304) – bare except swallows everything
try:
    usage = psutil.disk_usage(path)
    health_data["disk_usage"][path] = {
        "used_percent": round((usage.used / usage.total) * 100, 2),
        "free_gb": round(usage.free / (1024**3), 2),
        "total_gb": round(usage.total / (1024**3), 2)
    }
except:
    pass
```

### Root Cause

The disk-usage read block used a bare `except:` clause instead of a scoped
`except Exception` handler. The module already imports `logging` and defines a
module-level `logger = logging.getLogger("vscode-mcp")` (line 32), but the
catch block made no use of it.

## Fix Applied

Replaced the bare `except:` / `pass` with a typed `except Exception as e:`
clause that logs the failure at DEBUG level via the module logger. This preserves
the non-fatal intent (a single path failure should not abort the whole health
check) while surfacing errors for diagnostics.

```python
# AFTER
except Exception as e:
    logger.debug("Could not read disk usage for %s: %s", path, e)
```

`KeyboardInterrupt` and `SystemExit` are no longer swallowed, allowing the
process to terminate cleanly when requested.

## Impact

- Operators can now see disk-usage read failures in debug logs rather than
  having them silently discarded.
- No behaviour change for the happy path (all paths readable).

  sibling servers (e.g., `enhanced_mcp_server_phase2.py` HAO-359 fix).

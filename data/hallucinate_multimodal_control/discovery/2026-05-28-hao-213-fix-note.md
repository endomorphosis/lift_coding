# HAO-213 Fix Note

Date: 2026-05-28
Fingerprint: adf00600c6b3c9428483e38431de58a30fe49529
Status: resolved

## What was changed

`ipfs_accelerate_server_mp.py` line 102 had a bare `except Exception: pass` that silently
swallowed the connection error when trying to attach to an existing Plasma store.

The exception is **expected** when no store is running yet (the code immediately starts a new
one), but the silent `pass` made it impossible to diagnose genuine problems (wrong socket path,
permission errors, etc.).

## Fix

Bound the exception to `connect_err` and added a `logger.debug(...)` call that logs the socket
path and the exception message. This preserves the "try-then-start" flow while surfacing
diagnostic detail at DEBUG level.

## False-positive risk

Low. The scanner pattern `except Exception:` without binding or logging is a legitimate
maintenance smell. The fix is minimal and keeps the surrounding logic unchanged.
